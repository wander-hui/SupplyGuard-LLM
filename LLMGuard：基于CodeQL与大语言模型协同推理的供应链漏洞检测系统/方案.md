# 五、第一模块：依赖分析

这是供应链检测的基础。

---

## Java

解析：

```xml
pom.xml
```

提取：

```text
groupId
artifactId
version
```

例如：

```xml

```

---

## Python

解析：

```text
requirements.txt
```

例如：

```text
pyyaml==5.1
requests==2.20
```

---

## C++

解析：

```text
CMakeLists.txt
vcpkg.json
conanfile.txt
```

例如：

```text
yaml-cpp
openssl
```

---

输出统一格式：

```json
{
  "library":"fastjson",
  "version":"1.2.24"
}
```

# 七、第三模块：CodeQL分析

这里是整个系统核心。

---

## 目标

不要让 CodeQL 检测漏洞。

让 CodeQL 提取：

```text
Source

↓

Path

↓

Third-party API

↓

Sink
```

---

例如：

Python

```python
data=input()

obj=pickle.loads(data)
```

```json
{
 "source":"input",
 "sink":"pickle.loads"
}
```

---

Java

```java
request.getParameter()

↓

parseObject()
```

输出：

```json
{
 "source":"request",
 "sink":"parseObject"
}
```

# 八、第四模块：风险传播链切片

这是创新点之一。

---

原始调用链：

```text
Controller

↓

Service

↓

Helper

↓

Util

↓

DAO

↓

Fastjson
```

50多个节点。

---

压缩后：

```text
request

↓

service.query

↓

Fastjson.parseObject
```

---

生成：

```json
{
 "source":"request",
 "key_path":[...],
 "sink":"parseObject"
}
```

---

这样：

* Token下降90%
* LLM准确率提高
